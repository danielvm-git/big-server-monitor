## State management

No external state library (React built-ins only for v1):

```ts
// useServers.ts
function useServers(pollingMs = 5000) {
  const [servers, setServers] = useState<Server[]>([])
  useEffect(() => {
    const tick = () => GetServers().then(setServers)
    tick()
    const id = setInterval(tick, pollingMs)
    return () => clearInterval(id)
  }, [pollingMs])
  return servers
}
```

Similar hooks: `useHealthResults`, `useActivityLog`, `useLogs`, `useSettings`.

## New fields in the server row (ServBay-inspired additions)

### ServerRow — main line
- Runtime version badge next to process name: `node v20.11` in a muted monospace chip
- Tunnel URL pill (if detected): `→ abc123.ngrok.io` in blue, clickable to open in browser

### ServerRowExpanded — detail grid
Existing: PID, Binary path, Memory  
New additions:
- **Runtime version** — full string, e.g. `node v20.11.0 (darwin arm64)`
- **Local domain** — `myapp.local` as a clickable link (opens in browser)
- **Tunnel** — full public URL as a clickable link
- **Env vars** — key/value rows; safe keys show value, redacted keys show `***`; collapsed behind a "Show env (N)" toggle to avoid clutter

### HealthCheckSheet — result rows
Each row now shows **Protocol** pill alongside the status indicator:
- HTTP servers: `HTTP 200` / `HTTPS 302`
- Databases: `postgres` / `redis` / `mysql` badge — no status code, just latency + OK/timeout
- Protocol color: indigo for HTTP, slate for DB protocols

## Key interactions

| Interaction | Behavior |
|---|---|
| Click menubar icon | Toggle popover open/closed |
| Click server row | Expand/collapse detail (PID, memory, binary) |
| Click kill button (✕) | Open KillConfirmDialog |
| Click logs button | Open ServerLogsSheet for that server |
| Click "Copy for AI" | Call `GetLogsForAI(port)` → clipboard → show "Copied!" toast for 2s |
| Click "Health Check" | Open HealthCheckSheet |
| Click "Activity Log" | Open ActivityLogSheet |
| Click "Settings" | Open SettingsModal |
| Click "Test All Now" | Call `RunHealthCheck()` → update results |
| Click "Clear History" | Call `ClearHistory()` → empty list |
| Dark mode toggle | Toggle `[data-theme="dark"]` on `<html>` + persist to `localStorage` |
